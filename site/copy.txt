SECTION: THE WRONG NUMBER
-------------------------
H2: /usage shows you the wrong number.
Big: A 7-day limit holds about 2.8 days of full 5-hour sessions.
Fix: Spend every window to the wall and the week is gone by Wednesday night.

SECTION: HERE IS THE MATH
-------------------------
H2: Here is the math.
Lead: Four lines. That is the whole product.

pace    to_spend = week_left ÷ 5h_windows_left
        The sustainable token usage for this next 5hr period.
anchor  cap = burned ÷ used%
        Pinned to Claude's own /usage numbers every refresh.
burn    per line = (input + 5·output + 1.25·cache_write + 0.1·cache_read) × model_price
        Different models cost different
models  haiku ⅓ · sonnet 1 · opus 5⁄3 · fable 10⁄3
        Price-weighted. Refreshed daily from Anthropic's price sheet.

Footnote: A Fable token costs ten times a Haiku token, so maxx counts your tokens at what they actually cost.

SECTION: HOW TO READ THE BAR
----------------------------
H2: How to read the bar
Lead: Two rails: this session, and the week.

session: what you can safely spend now — your weekly budget paced across the 5-hour windows left.
week: tokens left in your 7-day limit, updated every second.
context: how full the context window is, plus model, branch, spend and cache-hit rate.

Animated notes (cycle under the demo rail):
- Under pace. Banking tokens.
- Steady. Right on pace.
- Over pace. Ease off.
- About to hit the 5-hour wall, where Claude stops you.

SECTION: INSTALL
----------------
H2: Install
Lead: One line in your terminal. Restart Claude Code. Done.
Hint 1: Prefer the plugin manager (skill only, no bar)? In Claude Code: /plugin marketplace add goodindustries/Maxx then /plugin install maxx@maxx.
Hint (terminals): Renders through the Claude Code statusline in plain ANSI, so it works wherever Claude Code does: Terminal.app, iTerm2, Ghostty, Warp, Alacritty, kitty.
Hint 2: Then type /maxx any time for total tokens, tokens/day, cache-hit rate and streak. Or /maxx session for how much to burn this window.

SECTION: EVERY MACHINE, ONE TALLY
---------------------------------
H2: Every machine, one tally.
Lead: Your quota is one pool, so maxx counts it as one: laptops, Claude Code cloud routines, claude.ai sessions. The installer claims a handle from your Claude login and starts tracking on its own. Use this form first if you want a particular name. One handle per Claude account.
Hint: Your own dashboards can subscribe too: webhooks push over / recovered / week-80 / week-90 / week-95 / runaway transitions (POST /api/u/<you>/webhooks), and GET /api/u/<you>/budget serves live availability, burn rate and top burners for polling.

SECTION: PRIVACY
----------------
H2: Your stuff stays yours
Lead: Nothing leaves your machine until you claim a handle. After that, maxx sends counts only: tokens, timestamps, model names. Never a prompt, never a message, never your code — the emitter cannot leak what it never reads. It is MIT licensed, so you can read the code.
